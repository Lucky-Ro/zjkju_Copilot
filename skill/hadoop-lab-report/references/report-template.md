### 表 1 —— 正文(6 行,每行一个整宽栏目)(**内容段落左对齐**)

**关键:每个栏目的标签和正文在同一个单元格里**(标签段后换行接正文)。填充时**定位到含该标签
的 `<w:tc>`,在标签段之后追加**内容段落 / 截图 / 代码块表格,**不要新建表格或破坏版式**。

| 行 | 单元格首段(标签) | 往该单元格追加什么 |
|---|---|---|
| R0 | 实验目的: | 各子任务【任务目的】汇总(分条) |
| R1 | 实验内容及实验器材: | 【任务环境】+【任务资源】(JDK/IDEA/Hadoop/MariaDB、安装包等,分条) |

> R2 是工作量大头。截图与代码块按教程步骤**顺序穿插**,使「读起来=照着教程做了一遍」。

## 学号替换规则(全局生效)

设 `sid3 = student_id_last3`(完整学号的后 3 位,下文记作 `NNN`,即 `<完整学号>` → 后3位 `NNN`)。
教程与你生成的内容里凡出现下列占位,**全部替换为 `sid3`**:
- 文案占位:「你的学号后3位」「替换为你学号后3位」「学号后三位」「<学号后3位>」等。
- 主机名:`nodea+你学号后3位` → `nodeaNNN`(同理 `nodeb`/`nodec`…)。
- 表名 / 库名 / 文件名 / 路径里嵌入的学号占位。
- HiveQL / SQL / XML 配置值里的学号占位。

替换在**执行阶段**(真正下发命令前)和**排版阶段**(写进报告前)都要做,保证「命令、输出、报告」三处一致。
替换前后各记一次,方便核对。**只替换占位,不要误伤教程里本就具体的示例值。**

## 内容改写口径(放轻松,像学生自己记实验笔记)

目标是让报告读起来像学生**自己记的实验笔记**:**短句、随手记、看图说话**,没有作文腔,自然就好、不用刻意。几个轻提示:

- **短句、动词领头就行**,一句话说清这步在做什么。像他写的:「启动 MariaDB,并设置开机启动」
  「执行以下 SQL 语句」「上传 Hive 安装包到 /home/hadoop 并解压」。
- **描述截图时,看图说话**——常用「可以看到…」「如图,…」「这里…」「发现…」。
  例:「可以看到环境变量指向了正确的位置」「这回 hive 库返回了预期的表」「发现并未返回预期中的表」。
  说清这张图证明了什么即可,一句话。
- **少用「我」,几乎不用作文式的第一人称长句**。像「我在 NodeA 上安装了 Hive,随后通过执行……命令
  完成了……的配置」这种就太「AI 味/刻意」了——能省则省,直接写动作和结果更自然。
- 偶尔带点真实的口语/小情绪是加分的(比如调通后来一句「破案了」),但别硬凑,顺其自然。
- 命令/HiveQL 放进 #FFF2CC 代码块,别在正文里复述;正文只留那一句说明 + 截图。
- 步骤说明可取自 `plan.json` 的 `step.text`,但**用你自己的话顺一遍**成上面的语气,别照抄教程的祈使原句。

### 排版铁律(实测自样例,生成器 `fill_report.py` 已固化,人工微调也照此)
- **对齐(两段式)**:
  - **首页表头(表0)个人信息**——姓名/学号/学院/班级/教师/地点/时间等**值单元格一律居中**
    (`jc=center`)。模板本是居中,但 python-docx `cell.text=` 会清成无对齐、退回左对齐,故须**显式补回居中**。
  - **后文正文(表1)**——实验目的/内容/过程/结论/总结里的所有内容段落**一律左对齐**(`jc=left`),
    **显式设 LEFT**,防止继承模板/样式里的居中。
- **顺序固定**:**短说明句(上) → 代码块 → 结果截图(下)**,逐步骤重复;**全程不写图注**
  (说明句已交代这张图证明了什么,图下不再加灰色斜体小字)。
- **截图**:**左对齐、满栏(≈15.5cm)、等比**。
- **说明句**:**无首行缩进、单倍行距、不加粗**;**只有子任务/小节标题加粗**。
- **代码块(黄色命令框)**:底色 #FFF2CC + `clear`、Consolas 五号(超长行降 9pt)、**细黑边框 `sz4 #000000`**、
  **宽度满栏(pct 100%,撑满整页文字区,不随命令长短缩放)**。
